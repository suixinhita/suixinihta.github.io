ROS的基本架构

  • 基本单位是节点（node)，是独立的可执行文件，可以是不同语言编写，也可以分布在不同的地方，但在整个系统中必须是唯一名字的；
  • 节点管理器(ROS master)，可以用来注册、命名节点，同时可以查找节点之间的连接，暂存节点在运行时的参数
  • 话题（topic),通信机制，用于节点之间传输数据，使用发布/订阅模型
  • 消息（message/msg)，是描述ROS中所使用消息类型的简单文本，具有一定的类型和数据结构，用与编程语言无关的.msg文件定义，编译形成的是对应的代码，存放在msg目录下
  • 服务（service/srv），同步通信系统，使用客户端/服务器模型，使用与编程语言无关的.srv定义，编译后同样形成对应的代码 ，包含请求和响应，存放在srv目录下

可以使用的数据类型如下：
int8, int16, int32, int64 (plus uint)* float32, float64 string time, duration other msg files variable-length array[] and fixed-length array[C]*

其中，消息和服务最大的差别在于其模型，服务主要用于逻辑处理，消息主要用于数据传输

  • 参数（parameter），全局参数，适合存储一些静态的参数，节点使用该服务来储存和检索运行时的参数
  • 功能包（package），ros中的基本单元，包含节源代码，配置文件等，其包含一个功能包清单，记录了其基本信息。

ROS文件系统

基本概念

  • packages:最基本的组织，里面可以有库、工具、可执行文件等
  • manifest：一个描述packages的文件，定义其之间的依赖关系，是xml文件，也
  • stacks：packages 的集合
  • stack manifest：描述stacks的文件

1 . 关于launch文件的写法:

<launch>

<group ns="turtlesim1">
<node pkg="turtlesim" name="sim" type="turtlesim_node"/>
</group>

<group ns="turtlesim2">
<node pkg="turtlesim" name="sim" type="turtlesim_node"/>
</group>

<node pkg="turtlesim" name="mimic" type="mimic">
<remap from="input" to="turtlesim1/turtle1"/>
<remap from="output" to="turtlesim2/turtle1"/>
</node>

</launch>

首先 前后的</launch>是表明这是个什么文件
中间的两段是创建了两个节点,并把其命名空间命名为12(ns是namespace的简写),然后在每个命名空间里都使用相同的节点turtlesim,并命名为sim
第三段是模仿，将所有topic的输入输出重命名，使得2模仿1

2 . 关于talker的c++写法

#include "ros/ros.h"
//是ros常用的头文件
#include "std_msgs/String.h"
//这个 前面是package名字/后面是message的名字
#include <sstream>
//这个头文件有点问题
/**
 * This tutorial demonstrates simple sending of messages over the ROS system.
 */
int main(int argc, char **argv)
{
  /**
   * The ros::init() function needs to see argc and argv so that it can perform
   * any ROS arguments and name remapping that were provided at the command line. For programmatic
   * remappings you can use a different version of init() which takes remappings
   * directly, but for most command-line programs, passing argc and argv is the easiest
   * way to do it.  The third argument to init() is the name of the node.
   *
   * You must call one of the versions of ros::init() before using any other
   * part of the ROS system.
   */
  ros::init(argc, argv, "talker");
  //初始化ros “里面的是最后得到的程序名称”
  /**
   * NodeHandle is the main access point to communications with the ROS system.
   * The first NodeHandle constructed will fully initialize this node, and the last
   * NodeHandle destructed will close down the node.
   */
  ros::NodeHandle n;
  //节点创建一个句柄
  /**
   * The advertise() function is how you tell ROS that you want to
   * publish on a given topic name. This invokes a call to the ROS
   * master node, which keeps a registry of who is publishing and who
   * is subscribing. After this advertise() call is made, the master
   * node will notify anyone who is trying to subscribe to this topic name,
   * and they will in turn negotiate a peer-to-peer connection with this
   * node.  advertise() returns a Publisher object which allows you to
   * publish messages on that topic through a call to publish().  Once
   * all copies of the returned Publisher object are destroyed, the topic
   * will be automatically unadvertised.
   *
   * The second parameter to advertise() is the size of the message queue
   * used for publishing messages.  If messages are published more quickly
   * than we can send them, the number here specifies how many messages to
   * buffer up before throwing some away.
   */
  ros::Publisher chatter_pub = n.advertise<std_msgs::String>("chatter", 1000);
  //在chatter topic上发布一个std_msgs/string消息 然后 1000是发布序列的大小 NodeHandle::advertise() 返回一个 ros::Publisher对象,它有两个作用: 1) 它有一个publish()成员函数可以让你在topic上发布消息； 2) 如果消息类型不对,它会拒绝发布。
  //chatter_pub是要求发出和接收是一样的，然后chatter是其topic名字 <这里面和头文件一样> 
  ros::Rate loop_rate(10);
  //指定的自循环的频率 
  /**
   * A count of how many messages we have sent. This is used to create
   * a unique string for each message.
   */
  int count = 0;
  while (ros::ok())
  /*
  ros::ok()返回false，如果下列条件之一发生：
    SIGINT接收到(Ctrl-C)
    被另一同名节点踢出ROS网络
    ros::shutdown()被程序的另一部分调用
    所有的ros::NodeHandles都已经被销毁
一旦ros::ok()返回false, 所有的ROS调用都会失效
  */
  {
    /**
     * This is a message object. You stuff it with data, and then publish it.
     */
    std_msgs::String msg;
    std::stringstream ss;
    ss << "hello world " << count;
    msg.data = ss.str();
    /*我们使用一个由msg file文件产生的‘消息自适应’类在ROS网络中广播消息。现在我们使用标准的String消息，它只有一个数据成员"data"。当然你也可以发布更复杂的消息类型。*/
    //事实上我不知道为什么这玩意要这样写 我直接写的
    ROS_INFO("%s", msg.data.c_str());
    //ROS_INFO和类似的函数用来替代printf/cout. 
    //这个我也不知道为什么，printf和cout亲测可用
    /**
     * The publish() function is how you send messages. The parameter
     * is the message object. The type of this object must agree with the type
     * given as a template parameter to the advertise<>() call, as was done
     * in the constructor above.
     */
    chatter_pub.publish(msg);
    //现在我们已经向所有连接到chatter topic的节点发送了msg
    ros::spinOnce();
    loop_rate.sleep();
    //调整时间用的 
    ++count;
  }
  return 0;
}

放一个自己的 已知std可以用的

#include <ros/ros.h>
#include <tryit/tryit_data.h>
#include <tryit/tryit_srv.h>
#include <cstdio>
#include <iostream>

using namespace std;

int main(int argc,char **argv)
{
    ros::init(argc,argv,"talker");
    ros::NodeHandle n;
    ros::Publisher msg_pub=n.advertise<tryit::tryit_data>("chatter",1000);
    ros::Rate loop_rate(10);
    int tot=0;
    while(ros::ok())
    {
        tryit::tryit_data msg;
        msg.txt="QAQ";
        msg.cnt=tot;
        cout<<"msg.txt="<<msg.txt<<endl;
        cout<<"msg.cnt="<<msg.cnt<<endl;
        
        msg_pub.publish(msg);
        ros::spinOnce();
        loop_rate.sleep();
        ++tot;
    }
    return 0;
}

3 . listener的c++写法

include "ros/ros.h"
#include "std_msgs/String.h"
/**
 * This tutorial demonstrates simple receipt of messages over the ROS system.
 */
void chatterCallback(const std_msgs::String::ConstPtr& msg)
//首先需要说明这里的string事实上我没搞懂，我是用的srv文件的文件名成功的 很奇妙
{
  ROS_INFO("I heard: [%s]", msg->data.c_str());
}
/*这是一个回调函数，当消息到达chatter topic的时候就会被调用。消息是以 boost shared_ptr指针的形式传输，这就意味着你可以存储它而又不需要复制数据*/
int main(int argc, char **argv)
{
  /**
   * The ros::init() function needs to see argc and argv so that it can perform
   * any ROS arguments and name remapping that were provided at the command line. For programmatic
   * remappings you can use a different version of init() which takes remappings
   * directly, but for most command-line programs, passing argc and argv is the easiest
   * way to do it.  The third argument to init() is the name of the node.
   *
   * You must call one of the versions of ros::init() before using any other
   * part of the ROS system.
   */
  ros::init(argc, argv, "listener");
  /**
   * NodeHandle is the main access point to communications with the ROS system.
   * The first NodeHandle constructed will fully initialize this node, and the last
   * NodeHandle destructed will close down the node.
   */
  ros::NodeHandle n;
  /**
   * The subscribe() call is how you tell ROS that you want to receive messages
   * on a given topic.  This invokes a call to the ROS
   * master node, which keeps a registry of who is publishing and who
   * is subscribing.  Messages are passed to a callback function, here
   * called chatterCallback.  subscribe() returns a Subscriber object that you
   * must hold on to until you want to unsubscribe.  When all copies of the Subscriber
   * object go out of scope, this callback will automatically be unsubscribed from
   * this topic.
   *
   * The second parameter to the subscribe() function is the size of the message
   * queue.  If messages are arriving faster than they are being processed, this
   * is the number of messages that will be buffered up before beginning to throw
   * away the oldest ones.
   */
  ros::Subscriber sub = n.subscribe("chatter", 1000, chatterCallback);
  /*告诉master我们要订阅chatter topic上的消息。当有消息到达topic时，ROS就会调用chatterCallback()函数。第二个参数是队列大小，以防我们处理消息的速度不够快，在缓存了1000个消息后，再有新的消息到来就将开始丢弃先前接收的消息。
NodeHandle::subscribe()返回ros::Subscriber对象,你必须让它处于活动状态直到你不再想订阅该消息。当这个对象销毁时，它将自动退订 上的消息。
有各种不同的NodeHandle::subscribe()函数，允许你指定类的成员函数，甚至是Boost.Function对象可以调用的任何数据类型。roscpp overview 提供了更为详尽的信息。*/
  /**
   * ros::spin() will enter a loop, pumping callbacks.  With this version, all
   * callbacks will be called from within this thread (the main one).  ros::spin()
   * will exit when Ctrl-C is pressed, or the node is shutdown by the master.
   */
  ros::spin();
  /*os::spin()进入自循环，可以尽可能快的调用消息回调函数。如果没有消息到达，它不会占用很多CPU，所以不用担心。一旦ros::ok()返回FALSE，ros::spin()就会立刻跳出自循环。*/
  return 0;
}

ROS 基本工具（命令）

需要强调的是，所有的ros命令都仅适用于环境变量ROS_PACKAGE_PATH中存在的目录路径,其可以添加任意目录路径，路径之间用”:”分开

对于ros命令，输入 $ tab $ 可以自动补全

在每次使用ros之前（也就是每次打开一个新的窗口的时候）（不过我应该是设置好了，可以不用写），都需要setup一下，具体代码如下：

source /opt/ros/melodic/setup.bash 或者 source ./devel/setup.bash

  1. rospack / rosstack 用于获取 packages和stack的信息(写在前面，意思是对这些做一些操作)

    • find [name] 查找路径
    • depends1(注意是123的1) [name] 查看依赖关系(指第一层的，在创建的时候写上去的依赖）
    • depends [name] 查看所有依赖关系，包括依赖的依赖

  2. roscd: 可以快速跳转到某个package或者stack的目录底下，也可以直接进入一个package或者stack的子目录下面 (eg. roscd roscpp/include 但事实上我的这个下面没有include 只有：cmake msg package.xml rosbuild srv 暂时不知道为什么)
    如果不带参数就是进入环境变量 ROS_ROOT的目录（但我操作后不是这样的，暂时不知道原因)
    参数是log的话是直接切换到ROS储存日志文件的目录。

  3. rosls 列出制定的package和stack的文件或目录

  4. roscreate-pkg 用于创建一个新的package 在后面先写package的名字 然后列出其依赖的其他package (一般会有roscpp rospy ros_msgs等）

  5. rosdep 安装ros系统依赖文件的工具 rosdep install [name]是命令 （出现了一个恐怖的事情，我这个没办法安装rxtools，我以防万一打算从github上手动clone了一个（事实上根本不会用））

  6. rosmake 对于package进行编译，同时按顺序编译其中的各个依赖 可以这样写 rosmake [name1] [name2]来编译多个

  1. catikn_make：建立一个catkin工作区

  2. roscore,是使用ros必须先运行的一个程序

  3. rosrun ,是运行节点使用的 可以用来重命名节点 如 rosrun turtlesim[位置] turtlesim_node[节点名称] __name:=my_turtle 就是将其重命名，如果不加入后面的__name:就不是重命名

  4. rosnode，用rosnode list可以显示出当前正在运行的节点 如果使用 rosnode info /[name] 就是询问详细信息

  5. 关于topic之间的关系 可以用rqt_graph来查看（但教程写的是rxgraph，我显示是没有这个命令 的，暂时不知道原因，怀疑与之前的rxtoos有关）

  6. rostopic ，用于查看topic相关信息的 （-h可以查看命令帮助）

  - echo [文件地址] 将在屏幕上显示topic的相关信息 （同时，官方教程存在一定差别，在melodic这个版本里面，发布topic的/turtle1/cmd_vel） 
  - list 可以查看正在运行的topic的信息（同样-h也可以看帮助）
  - type 可以查看一些topic的类型
  - pub 可以用来publish一些数据 格式 rostopic pub [topic] [msg_type] [args]

  1. rosmsg 可以用于查看详细信息，使用show [name]的格式

  2. rqt_polt 用于画图 用法 [topic] [参数] 具体可以查阅-h

  3. rosservice ，可以查看service相关的信息

    • list 列出现有的service
    • type 判断service的类型
    • call 使用service （clear相当于update)

  4. rosparam 可以在服务器上存储一些参数，使用YAML语法

    • list 列出所有参数
    • set 改变参数（之后需要 call clear)
    • get 获得参数值 （get / 是获取全部vu参数值的意思）
    • load 从文件中提取参数
    • dump 将参数提取到某个文件 (eg. rosparam dump [file_name]
      rosparam load [file_name] [namespace])

  5. rosed ，属于一种快捷键，可以直接通过package名来得到文件，用法rosed [package_name] [filename]

  6. rqt_console和rqt_logger_level属于日志框架的一部分，用来显示节点的输出信息，后者可以修改日志等级（DEBUG，WARN一类）启动方法是rosrun rqt_console rqt_console（rqt_logger_level同理）日志的等级从低到高依次是Fatal、Error、Warn、Info、Debug

  7. roslaunch 可以用来启动定义在launch文件里的多个节点，用法roslaunch [package] [filename.launch]

  8. roscp 复制，用法 roscp [package_name] [file_to_copy_path] [copy_path]
